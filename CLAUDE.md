# CLAUDE.md

## Project Overview

Autonomous TikTok content engine with a **triple flow system** — generates photorealistic, animated, AND emoji overlay slideshow posts, posts to TikTok via Blotato, tracks performance via ScrapeCreators, and self-improves via A/B testing. Runs via Claude Code `/loop`.

## Commands

```bash
npm run cycle          # Full cycle: 3 flows × active accounts
npm run research       # Virlo trend research only
npm run generate       # Content generation only
npm run analytics      # Pull performance metrics
npm run optimize       # Run A/B optimizer
npm run refresh        # Daily refresh: analytics + optimizer + research (all data files)
npm run refresh:quick  # Daily refresh without Virlo research (saves API credits)
```

All scripts use `tsx` (no build step). Entry point: `main.ts`. Pipeline scripts in `scripts/`.

## Triple Flow System

### Flow 1: Photorealistic
- Cinematic, real-looking photographs (Arri Alexa / Sony A7 quality)
- Consistent character per post (same face, hair, outfit)
- Stylish fonts (italic, cursive, handwritten) — never plain
- Cinematic color grading (warm golden or cool moody)
- Shallow depth of field, study-related scenes

### Flow 2: Animated
- Different animation style EVERY post (15+ styles rotating)
- Styles: Pixar 3D, Stop Motion, 90s Cartoon, Watercolor, Anime, Chalk, Pop Art, Pixel Art, etc.
- Consistent character across slides within one post
- Warm, colorful, expressive characters with big eyes

### Flow 3: Emoji Overlay
- Illustrated/animated characters (shares animation styles with Flow 2)
- **Emoji reaction bubbles** overlaid on each slide (semi-transparent, top-right corner)
- **Narrative arc structure:**
  - Slide 1: Hook (🤔), Slides 2-3: Problem (😰), Slides 4-6: Tips (💡), Slide 7: Resolution (🔥), Slide 8: CTA (👉)
- Character expressions change per slide type (anime-style exaggerations: sweat drops, sparkle eyes)
- Background decided by AI based on topic (not from fixed scene list)
- Character visually progresses through the story
- Expression mapping:
  - Hook → curious, raised eyebrow, leaning forward
  - Problem → frustrated, head in hands, slouched
  - Tips → focused, "aha" moment, eyes lit up
  - Resolution → confident smile, standing tall
  - CTA → warm smile, inviting gesture

### All Flows
- Text in top 60-70% of image (TikTok safe zone)
- Stylish fonts only (italic/cursive/handwritten)
- Last slide = pre-made CTA image (MinuteWise App Store)
- Content generated by Gemini AI (fresh per post), CSV templates as fallback

## Architecture

**Pipeline:** `fetch_trends.ts → text_overlay.ts → generate_images.ts → post_to_tiktok.ts → pull_analytics.ts → optimizer.ts`

Orchestrated by `main.ts`. Each flow run produces 1 post per active account. Default posting path is `draft` (TikTok drafts via Blotato). Use `--path=direct` to publish directly.

## Slide Blueprint (from SOUL.md)

1. **Hook** — Pattern interrupt, bold statement challenging beliefs
2. **Knowledge Gap** — Something viewer didn't know they didn't know
3. **Value Slides** — One gold nugget per slide, screenshotable
4. **MinuteWise** — Natural integration, solves a pain point mentioned earlier
5. **Emotional** — Social proof / "send to friend" trigger
6. **CTA** — Pre-made MinuteWise app store image

## Content Rules

- ALL posts follow `config/caption_templates.csv` patterns
- ALL posts naturally mention MinuteWise in at least one slide
- Hashtags: 20 per post — 5 trending (Tier 1) + 7 niche (Tier 2) + 5 ultra-niche (Tier 3) + 3 branded/topic (Tier 4). Always include #MinuteWise. See `data/HASHTAG-BANK.md` for full list.
- Captions: hook + value line + "Save for exam week" + hashtags
- ALL posts as DRAFTS — never auto-publish
- `autoAddMusic: "yes"`, `video_made_with_ai: true`

### Autoresearch (program.md)
Autonomous content experimentation inspired by [karpathy/autoresearch](https://github.com/karpathy/autoresearch).
- **Loop:** Every 6 hours
- **Evaluate:** Experiments older than 24h using ScrapeCreators per-post metrics
- **Metric:** Save rate (primary), views (secondary)
- **Action:** Keep winner → update FORMAT-WINNERS. Discard loser.
- **Generate:** Design next experiment (change 1 variable), generate 3 posts, post as drafts
- **Log:** Everything to `data/results.tsv`
- **Rule:** NEVER STOP. Human may be sleeping.
